# Module 7: Polish & Ship

**⏱ Time: ~60 minutes | 🔴 Difficulty: Advanced | 💻 Coding: Some**

Good enough isn't good enough. Let's make it great.

---

## What You'll Learn

- **Error handling** — What to do when things go wrong (they will)
- **Edge cases** — The weird questions and scenarios your agent needs to handle
- **Performance** — Making your agent faster and more efficient
- **Documentation** — Writing instructions so others can use your agent
- **Publishing** — Sharing your agent as a skill or template on ClawHub
- **Open source practices** — README, LICENSE, CONTRIBUTING, versioning

## Module Outline

### Lesson 1: Bulletproofing
- Common failure modes and how to handle them
- Graceful degradation — what happens when a skill is down?
- Testing strategies: the "annoying friend" test
- Logging and debugging

### Lesson 2: User Experience
- First impressions matter — the onboarding conversation
- Help commands and self-documentation
- Error messages that actually help
- The "mom test" — can someone non-technical use your agent?

### Lesson 3: Documentation
- Writing a README that makes people want to try your agent
- Installation instructions that actually work
- Screenshots and examples
- The CONTRIBUTING.md template

### Lesson 4: Ship It
- Publishing to ClawHub
- GitHub repo setup
- Versioning your agent
- Getting feedback and iterating

## Challenge Preview
Polish your Module 5 project to production quality: handle 10 edge cases, write documentation, and publish it to ClawHub or GitHub.

## Prerequisites
- Completed Modules 1-6
